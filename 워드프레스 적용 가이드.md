# LJ VIEW 홈 — 워드프레스 적용 가이드

## 프로젝트 구조 (독립 Netlify 사이트)

이 폴더(`ljview-homepage`)는 "엘제이뷰 강의 프로그램"(`ljview-course-app`)과 완전히 분리된 독립 프로젝트예요.
강의 프로그램 쪽 코드/DB와는 전혀 섞이지 않고, 노션 통합 토큰도 별도("엘제이뷰 홈페이지")로 씁니다.

```
ljview-homepage/
  public/
    index.html          — 홈 페이지 전체 (HTML+CSS+JS 한 파일, 워드프레스에 붙여넣을 대상)
    assets/
      lj-view-logo.png        — 컬러 로고 (밝은 배경용, 헤더에서 사용)
      lj-view-logo-light.png  — 화이트 로고 (어두운 배경용, 푸터에서 사용)
      hero.png                 — 히어로 배경 이미지
  netlify/functions/    — 노션 연동 공개 API 3개 (site-partners / site-reviews / site-contact)
  lib/notion.js          — 위 3개 함수가 쓰는 노션 클라이언트
  .env                   — 로컬 전용 시크릿(git에 안 올라감)
```

## ⚠️ 올리기 전에 딱 하나만 확인 — API_BASE

`public/index.html`의 `<script>` 맨 위에 이런 줄이 있어요.

```js
var API_BASE = 'https://YOUR-NETLIFY-SITE.netlify.app';
```

이 폴더(`ljview-homepage`) 자체를 Netlify에 배포한 뒤, 그 사이트 주소로 이 한 줄만 교체하면 됩니다.
**지금은 "엘제이뷰 강의 프로그램" 쪽 Netlify 크레딧이 소진돼 있어서(2026-08-02 이후 복구 예정) 이 프로젝트도 아직 배포 전이에요.**
그 전까지는 로고 마퀴는 기본 목록으로, 후기는 예시 카드로, 문의폼은 실패 메시지가 뜨는 상태로 안전하게 동작합니다(사이트가 깨지진 않음).

## 워드프레스에 올리는 법 (권장 순서)

1. **이 폴더를 Netlify에 배포** (크레딧 복구 후) → 실제 사이트 주소 확인
2. **API_BASE 교체** (위 항목)
3. **이미지 업로드**: 워드프레스 관리자 → 미디어 → `public/assets/` 안의 이미지 업로드
4. **새 페이지 생성**: 템플릿을 "빈 템플릿(Blank/Canvas)"으로 선택
   - 테마에 빈 템플릿이 없으면 Elementor Canvas 또는 "Blank Slate" 플러그인 사용
5. **커스텀 HTML 블록**에 `index.html`의 `<body>` 안 내용 + `<style>` + `<script>` 붙여넣기
6. **이미지 경로 교체**: `assets/...` → 업로드된 미디어 URL로 바꾸기 (헤더 로고, 히어로, 푸터 로고 총 3곳)
7. 홈으로 지정: 설정 → 읽기 → 정적인 페이지 → 홈페이지로 선택

## 노션 연동 (로고·후기·문의) — 로컬 테스트로 이미 검증됨

| 기능 | 노션 DB | 관리 방법 |
|---|---|---|
| 위드 엘제이뷰 로고 마퀴 | **위드 엘제이뷰 로고** | 기관명 입력 + 로고 파일 업로드 + 노출 체크만 하면 자동 반영. 로고 없으면 텍스트로 표시 |
| Reviews 후기 카드 | **홈페이지 후기** | 후기내용/작성자·기관/분류/사진/노출 채우면 자동 반영. 실제 후기가 하나도 없으면 예시 카드가 그대로 보임 |
| 문의폼 제출 | **고객 요청 관리** (기존 DB) | 제출 즉시 새 행 생성, 상태=신규로 들어옴. 문의구분은 강의/홍보대행/컨설팅/디자인 → DB의 요청유형 옵션에 자동 매핑 |

세 DB 모두 "엘제이뷰 홈페이지"라는 별도 노션 통합에만 연결돼 있어요(학생 개인정보가 있는 강의 프로그램 통합과 분리). 워드프레스에는 노션 토큰이 전혀 들어가지 않고, 전부 이 프로젝트의 Netlify 함수 뒤에 숨겨져 있습니다.

## 블로그 자동 연동
- `index.html`의 JS가 `/wp-json/wp/v2/posts?per_page=3&_embed=1`을 호출해 **최신 글 3개**를 자동으로 채웁니다.
- 같은 워드프레스 도메인에 올리면 별도 설정 없이 작동합니다.
- 연동 실패 시엔 예시 카드 3개가 그대로 보입니다(깨지지 않음).

## 히어로 이미지 관련 참고
- 현재 히어로는 인물 사진이 아니라 브랜드 오브젝트 플랫레이 이미지(`hero.png`)를 씁니다 — 밝은 이미지라 오버레이/텍스트 색이 어두운 톤으로 세팅돼 있어요.
- 나중에 다시 어두운 사진(인물 등)으로 바꾸고 싶다면, `.hero-overlay` 배경과 `.hero h1`/`.hero-sub`/`.hero-eyebrow .txt`/`.btn-ghost`/`.hero-scroll` 색상을 밝은 색(흰색 계열)으로 되돌려야 합니다. 지금은 전부 어두운 텍스트 톤으로 맞춰져 있어요.

## 콘텐츠 교체 체크리스트
- [ ] About 대표 사진 (누끼 PNG, `founder.png`로 추가하고 주석 처리된 `<img>` 활성화)
- [ ] 스탯 숫자 확인 (10년+/300건+/20건+/500명+) — 실제 수치 맞는지 확인
- [ ] 서비스 카드 4개의 `href="#"` → 상세 페이지 주소로 교체 (새 창으로 열림)
- [ ] 노션 "위드 엘제이뷰 로고" DB에 로고 파일 업로드
- [ ] 노션 "홈페이지 후기" DB에 실제 후기 입력 (현재 비어있음 → 예시 카드로 대체 노출 중)

## 후기 섹션 사진 관리 (마소니)
- 노션 "홈페이지 후기" DB에서 사진 필드를 채우면 자동으로 사진 카드로, 비우면 텍스트 카드로 렌더링됩니다.
- `분류` 값(강의/홍보대행/디자인)으로 탭 필터가 자동 적용됩니다.
- 사진 세로 비율은 자유 — 마소니 그리드라 높이가 달라도 자연스럽게 배치됩니다.

## 로컬 개발 서버 실행
```
cd ljview-homepage
npm install   # 최초 1회
netlify dev
```
`public/`이 정적 파일로, `netlify/functions/`가 API로 같은 서버(기본 8888)에서 함께 뜹니다.
